Workflow

So how where these projects created?

All projects are made in 4 steps, some smaller than others.

  1. The first step is to draw a sketch on paper. Some doodling just to get the idea of the concept you're trying to build. With a client involved, this is a perfect way to see if both parties have understood the task ahead.
  2. The next step is to convert it into a mockup on the computer. This is often more extensive than the sketching. It is easier to continue the work in Photoshop when having a base of sketches and mockups.
  3. The third step is where the magic happens. Here, the sketches and mockups come to life! This step is usually recursive and requires quite a few retakes before it gets right.
  4. The fourth and final step is slicing and converting the PSD file from previous step into a website.

The projects listed below are all personal where I did all the work. There was no client involved.

Dirty Chat

Dirty chat is a chat service where people can in a quick and easy way connect to other people through their browser. Everyone connect to the same chat channel and can freely chat with each other. All conversation is open for all visitors.

This application is built in Python with the Django framework. The database consists of MongoDB. The frontend is built with HTML5, CSS3, javascript and AJAX.

There is no demo available online, but you can view the source code by checking out my repo at git.bluecap.se

Backgammon

This project is a webbased Backgammon game that can be played by two people. The game was built to test the limits of HTML5 and CSS3 capability in the browser.

The game is built in HTML5, CSS3, canvas and Javascript.
